1. 잘 부탁하는 5원칙

결과가 안 좋으면 십중팔구 맥락이 부족해서입니다. 송무 지시서 쓰듯 자연어로 분명하게 부탁하세요.

① 맥락을 먼저
모델은 내 사건을 모릅니다. 맥락이 없으면 일반론·환각으로 메웁니다.
✗ "대여금 답변서 써줘" → 교과서 템플릿
✓ 사건폴더를 읽히고 "이 사건 기준으로" → 그 사건 답변서
② 역할·입장 지정
같은 사실도 입장 따라 강조점이 달라집니다.
✓ "원고(임차인) 대리인 입장에서" — 안 주면 어정쩡한 중립 글
③ 산출물 형식 명시
표·항·분량·저장파일을 못박으면 매번 같은 결과(재현성↑).
✓ "호증번호/서증명/입증취지/작성일 열로 표" → 증거설명서에 바로 붙음
④ 한 번에 다 말고 단계로
작업을 쪼개면 단계별 정확도가 오르고, 중간에 교정할 수 있습니다.
✓ 쟁점 추출 → 목차 → 초안 — "준비서면 다 써줘"보다 안전
⑤ 틀리면 되물어라
모델은 추론을 설명·수정할 수 있습니다. 단, 최종 검증 책임은 변호사.
✓ "이 인용 출처 표시하고, 불확실하면 표시해"
⚖️ 부족한 프롬프트 → 좋은 프롬프트 (Before / After)
✗ "이 사건 답변서 써줘"
맥락·입장·형식이 다 빠짐 → 일반 템플릿
✓ "이 폴더(차용증·문자·입금내역) 다 읽고 · 피고 대리인 입장에서 · 변제·소멸시효 항변 중심으로 · 쟁점별 항 나눠 1~2페이지, 답변서.md로 저장"
맥락 + 입장 + 형식을 다 박음 → 그 사건 답변서
좋은 프롬프트 = 맥락 + 입장 + 형식 · 나머지는 대화로 교정

2. 실사례에서 뽑은 9가지 노하우

과거 Claude Code 세션 약 220개에서 반복된 실패와 성공 패턴입니다. 실무에서는 "잘 쓰기"보다 "틀릴 만한 곳을 먼저 잠그기"가 더 중요합니다.

상황자주 생긴 문제예방 프롬프트
판례·법령 인용없는 사건번호, 폐기된 법리, 주문/신청취지 오독실제 조회된 자료만 쓰고, 미확인은 "미확인"으로 표시. 주문·이유·인용액을 먼저 대조
서면·내용증명 수정발신인/수신인, 원고/피고 입장 뒤바뀜발신인=A, 수신인=B, 우리 입장은 ○○라고 당사자 구도를 먼저 고정
견적·신청서 금액실비·부가세·산정사유 누락, 빈칸 방치체크리스트를 먼저 만들고 누락 자가점검 후 산식·가정과 함께 직접 채우기
협업 편집사람이 고친 최신본을 AI가 옛 버전으로 덮어씀수정 전 디스크 최신본 재읽기, 보존할 문단 확인, 변경요약 출력
문체 윤문LLM 말투, 공격적 어조, 준비서면답지 않은 도식금지표현·톤·형식을 먼저 주고, 전체 재작성보다 선택 문단만 핀포인트 수정
법률 리서치결론부터 쓰고 근거를 나중에 끼워 맞춤정보원 우선순위: legal-books → korean-law → beopgoeul-search → 웹, 부족하면 "부족" 표시

명단·분류정렬·제외기준·경계사례를 뒤늦게 정함정렬키, 컷오프, 제외조건, 경계인물 처리 기준을 먼저 지정

3. 업무별 '잘 부탁하는 법' (환경·도구 × 예시 프롬프트)

앱 자체를 부르는 것이 아니라, Claude Code 안에서 폴더·MCP·스킬·워크플로우를 업무에 맞게 쓰는 예시입니다.

1 사건 파악 — Claude Code(사건 폴더 기준)
이 사건폴더 전부 읽고 ① 사실관계 시간순 표, ② 쟁점 목록,
③ 우리(피고)에게 불리한 점 순으로 정리해줘.
2 법령 조사 — korean-law(조문 원문)
주택임대차보호법 제6조의3 현행 조문 원문과
직전 개정 전 조문을 표로 대비해줘.
3 판례 검색 — beopgoeul-search(무료)·lbox(유료 직접)
beopgoeul-search로 임차인 원상회복의무 범위에 관한 대법원 판결을 찾아
사건번호·선고일·판시사항을 표로, 출처 URL까지.
※ 판결번호는 1차 후보일 뿐 — 인용 전 반드시 진본 확인.
4 내 판단 검증 — legal-books(교과서 대조)
'일부변제는 채무 전부의 묵시적 승인으로 시효가 중단된다'는
내 판단이 통설·판례와 맞는지 legal-books에서 대조하고,
어긋나거나 예외가 있는 부분만 짚어줘.
5 과거 사건 참고 — case-records
유사한 대여금 사건에서 우리가 변제·시효 항변을 어떻게 구성했는지,
법원이 어떻게 판단했는지 과거 기록에서 찾아줘.
6 준비서면 — jurisupport brief-protocol
jurisupport의 brief-protocol 흐름으로 준비서면을 만들자.
인테이크 → 사건기록 분석 → 쟁점표 → legal-books/korean-law/beopgoeul-search 검증
→ 목차 승인 → MD 초안 → 인용 검증 순서로 진행하고,
각 단계가 끝나면 멈춰서 확인받아.
7 제출 전 모의변론 — jurisupport mock-hearing
jurisupport의 mock-hearing 방식으로 이 준비서면을 검토해줘.
청구권규범 카드, 요건사실 매트릭스, 입증책임 지도,
항변·재항변 트리, 유리/불리 판결 유추·구별 메모를 만들고,
제출가능/보강/재구성/출구 중 평결과 보강 과제를 줘.
8 당사자·금액·버전 검수 — 실사례 체크
작성한 초안에서 ① 원고/피고·발신/수신이 뒤바뀐 곳,
② 금액·날짜·사건번호가 원자료와 다른 곳, ③ 빈칸이나 추정 표현,
④ 최신 파일을 반영하지 않은 흔적을 표로 점검해줘.
9 사건관리·기일 — JuriSupport / case-index
JuriSupport MCP가 있으면 사건·기일·할일을 조회하고,
없으면 case-index의 _index.csv에서 2주 이내 기일이 있는 사건을 찾아줘.
사건번호 / 사건명 / 진행단계 / 다음기일 / 오늘 할 일을 표로 정리해줘.
※ JuriSupport 미사용자는 CSV 한 파일이 최소 사건관리 장부가 됩니다. JuriSupport 연동자는 JuriSupport가 정본, CSV는 백업·오프라인 뷰입니다.

4. 내 법률 서재(legal-books) — 무엇을 넣어도 되나

  1. 판례·결정문 전문 (저작권법 §7 3호 보호 제외) — 가장 안전
  2. 법령·행정규칙 전문 (§7 1·2호)
  3. 공공기관 무료 해설·실무자료 (공공저작물 자유이용 §24-2)
  4. 변호사 본인 저작물(서면·메모·블로그) — 가장 깨끗 + 사무소 노하우 누적
  5. CC 라이선스·저작권 만료 고전
⚠️ 상용 교과서·단행본 — 본인 업무용 참고·검색은 가능하지만, DB(복제물)를 남에게 배포·공유하는 것은 위험. (개인 이용 ≠ 배포)

5. 검증은 변호사 (가장 중요)

AI의 가장 큰 위험은 그럴듯한 가짜 판결번호(환각)입니다. 제출 전 반드시:

  1. 인용 판결번호 실존 확인 — beopgoeul-search(법고을) / lbox에서 직접 검색
  2. 직접인용("…")은 원문과 글자 단위 대조
  3. 법령 조문번호는 korean-law / law.go.kr로 재확인
  4. 청구액·인용액·신청취지·주문을 서로 구분해서 확인
  5. 사건번호·기일·당사자명처럼 식별자는 출처와 1:1로 대조
