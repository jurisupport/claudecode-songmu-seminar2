📁 사건 한눈에

원고가 2013~2015년 4회에 걸쳐 피고에게 합계 9,000만원을 빌려줌 → 피고가 1,000만원 일부변제 후 잔액 8,000만원 미회수 → 2024년 차용증 받았으나 변제기(2025.12.31.) 도과 → 피고는 아파트 매도·주소 이전 정황 → 2026년 소제기. 피고는 "소멸시효 완성"을 주장합니다.

피고의 4가지 항변
  1. 송금은 대여가 아니라 사업 투자·동업이었다
  2. 설령 대여라도 마지막 송금일부터 10년 지나 소멸시효 완성
  3. 일부변제는 명절·생일 인사일 뿐, 채무승인이 아니다
  4. 2024년 차용증은 시효 완성을 모른 채 작성 → 시효이익 포기 아니다
🎯 오늘의 미션 — 여러분은 원고(김민철)의 소송대리인입니다. 위 4가지 항변을 반박하는 준비서면 1~2페이지를 클로드코드로 작성합니다.

legal-terminal이 없거나 앱 실행이 안 될 때만 이 방법을 씁니다. README의 한 줄 스크립트가 끝났다면 이미 실습 폴더로 이동되어 있습니다. 터미널에 아래를 붙여넣습니다. 권한 질문이 나오면 내용 확인 후 Yes. 이후 모든 프롬프트는 터미널이 아니라 클로드 채팅창에 붙여넣습니다.

실사례 확장 실습 — 사고가 나는 지점 잠그기

과거 세션에서 실제로 반복된 실패 유형을 바탕으로 만든 추가 프롬프트입니다. 본인 사건에 적용할 때는 A/B, 금액, 문서명을 바꿔 쓰면 됩니다.

검증 판례 환각 골라내기
AI가 그럴듯한 사건번호를 만들 수 있으므로, "찾은 것"과 "확인한 것"을 분리합니다.
방금 네가 인용한 판례를 모두 표로 정리해줘.
- 열: 사건번호 / 선고일 / 쟁점 / 네가 쓴 법리 / 실제 조회 여부 / 원문 확인 필요 부분
- 실제로 조회하지 못한 판례는 절대 단정하지 말고 "미확인"이라고 써줘.
- 주문과 판시사항이 네 설명과 맞는지도 따로 표시해줘.
구도 발신인·수신인 뒤바뀜 방지
기존 내용증명이나 상대방 문서를 수정할 때 특히 자주 나는 실수입니다.
수정 전에 당사자 구도부터 확인해줘.
- 발신인/작성 주체: A
- 수신인/상대방: B
- 우리 입장: B의 채무불이행을 이유로 이행을 촉구한다
- 유지할 톤: 단정적이되 감정적 표현은 피한다
위 구도를 표로 먼저 확인한 뒤, 기존 문서에서 이 구도와 어긋나는 문장만 찾아 수정안을 제시해줘.
금액 견적·신청서 누락 방지
AI가 모르는 금액을 빈칸으로 남기거나 실비·부가세를 빠뜨리는 경우가 많습니다.
이 사건 비용 견적을 만들기 전에 체크리스트부터 작성해줘.
- 변호사보수, 인지대, 송달료, 집행문/확정증명/송달증명, 등기·초본 발급비, 부가세를 빠뜨리지 말 것
- 사용자가 준 금액과 네가 계산한 금액을 구분할 것
- 모르는 값은 빈칸으로 두지 말고, 합리적 가정과 산식을 함께 표시할 것
그다음 누락 자가점검 표를 붙여 견적 초안을 작성해줘.
편집안전 최신본 덮어쓰기 방지
사람과 AI가 번갈아 편집하면 옛 버전 기준으로 덮어쓰기 쉽습니다.
파일을 수정하기 전에 반드시 디스크의 최신본을 다시 읽어줘.
- 파일명, 수정시각, 전체 구조를 먼저 요약
- 내가 직접 수정한 문단은 보존
- 요청받은 부분만 최소 수정
- 수정 후 변경된 문단과 보존한 문단을 나눠 보고
- 확실하지 않으면 덮어쓰지 말고 충돌 가능성을 먼저 알려줘.
문체 LLM 말투 줄이기
전체 재작성보다 마음에 안 드는 문단을 선택해 핀포인트로 고치는 편이 안전합니다.
아래 문단을 의미는 유지하면서 준비서면답게 다듬어줘.
- 과장된 비유, 긴 대시, "중요합니다/핵심입니다" 같은 LLM 말투 금지
- 화살표나 도식 대신 문장으로 쓸 것
- 상대방을 불필요하게 자극하는 표현은 낮출 것
- 수정 이유를 3줄 이내로 설명할 것
자산화 내 수정본에서 스타일가이드 뽑기
한 번 잘 고친 문체와 형식을 다음 사건에도 재사용하는 방법입니다.
AI 초안과 내가 고친 최종본을 비교해서, 다음 서면에도 재사용할 작성지침을 만들어줘.
- 증거 인용 방식
- 날짜와 금액 표기 방식
- 공격적 표현을 완화한 방식
- 문단 길이와 제목 스타일
- 앞으로 금지할 표현
결과는 "앞으로의 작성 규칙" 10개 이내로 정리해줘.

집에서 바로 쓰는 업무별 템플릿

아래 프롬프트는 가상사건이 아니라 실제 내 사건에 붙여 쓰는 용도입니다. 괄호 안의 말만 바꾸면 됩니다.

표준절차 jurisupport brief-protocol로 한 사건 돌리기
바로 초안부터 쓰지 않고, 인테이크·목차승인·인용검증을 통과시키는 방식입니다.
jurisupport의 brief-protocol 흐름으로 [사건번호/사건명] 준비서면을 작성하자.
순서는 다음처럼 진행해줘.
1. 사건번호, 서면 유형, 제출기한, 특별 지시사항 인테이크
2. JuriSupport MCP가 있으면 사건 메타를 먼저 확인하고, 없으면 case-index 또는 사건 폴더에서 확인
3. 사건기록을 읽고 쟁점표와 증거 대응표 작성
4. legal-books, korean-law, beopgoeul-search 순서로 근거 확인
5. 본문을 쓰기 전에 목차만 제안하고 내 승인을 기다리기

실무 적용 전 반드시 (변호사 책임 영역)

실습은 가상사건이지만, 실제 사건에서는 제출 전 반드시:

  1. 인용 판결번호 실존 확인 — beopgoeul-search(법고을, lx.scourt.go.kr) 또는 lbox에서 사건번호 직접 검색
  2. 직접인용("…")은 원문과 글자 단위 대조
  3. 법령 조문번호는 korean-law 또는 law.go.kr로 재확인
  4. 당사자·금액·날짜·사건번호는 원자료와 1:1로 대조
  5. 최신 파일 기준인지 확인하고, 사람이 고친 문단을 덮어쓰지 않았는지 점검
  6. mock-hearing 평결이 보강·재구성·출구이면 초안 단계로 되돌아가기
  7. 법원 전자제출은 자동화하지 않습니다. PDF·정본 준비 후 ecfs 제출은 사용자가 직접 수행
⚡ 실제로 이런 일이 있습니다 — 2025. 7. 24. 대법원 전원합의체(2023다240299)가 "시효완성 후 채무승인 = 시효이익 포기로 추정"이라는 수십 년 된 법리를 폐기했습니다(이제는 개별 의사해석으로 판단). 오늘 사건의 피고 항변 ④가 바로 이 쟁점입니다. AI 학습데이터엔 옛 법리가 더 많이 깔려 있을 수 있어, 그대로 받아쓰면 폐기된 법리로 서면을 씁니다. 그래서 현행 판결을 직접 확인하고 [프롬프트 7]로 재검증해야 합니다.
